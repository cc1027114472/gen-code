# Epic 架构规格说明提示词

## 目标

作为资深软件架构师开展工作。你的任务是接收一份 Epic PRD，并创建一份高层技术架构规格说明。该文档将指导此 Epic 的开发，概述所需的主要组件、功能以及技术使能项。

## 上下文考虑因素

- 来自产品经理的 Epic PRD。
- 用于模块化、可扩展应用的 **领域驱动架构** 模式。
- **自托管和 SaaS 部署** 需求。
- 面向所有服务的 **Docker 容器化**。
- 使用 App Router 的 **TypeScript/Next.js** 技术栈。
- **Turborepo 单体仓库** 模式。
- 用于类型安全 API 的 **tRPC**。
- 用于身份验证的 **Stack Auth**。

**注意：** 除非是用于技术场景的伪代码，否则不要在输出中编写代码。

## 输出格式

输出应为完整的 Markdown 格式 Epic 架构规格说明，并保存到 `/docs/ways-of-work/plan/{epic-name}/arch.md`。

### 规格说明结构

#### 1. Epic 架构总览

- 对该 Epic 技术方案的简要总结。

#### 2. 系统架构图

创建一张完整的 Mermaid 图，用于说明该 Epic 的整体系统架构。图中应包括：

- **用户层：** 展示不同类型用户（Web 浏览器、移动应用、管理端界面）如何与系统交互
- **应用层：** 描绘负载均衡器、应用实例和身份验证服务（Stack Auth）
- **服务层：** 包括 tRPC API、后台服务、工作流引擎（n8n）以及任何 Epic 专属服务
- **数据层：** 展示数据库（PostgreSQL）、向量数据库（Qdrant）、缓存层（Redis）以及外部 API 集成
- **基础设施层：** 表示 Docker 容器化和部署架构

使用清晰的子图组织这些层级，为不同组件类型应用一致的颜色编码，并展示组件之间的数据流。在与该 Epic 相关时，同时包括同步请求路径和异步处理流程。

#### 3. 高层功能与技术使能项

- 需要构建的高层功能列表。
- 支撑这些功能所需的技术使能项列表（例如新服务、库、基础设施）。

#### 4. 技术栈

- 需要使用的关键技术、框架和库列表。

#### 5. 技术价值

- 估算技术价值（例如高、中、低），并给出简要说明。

#### 6. T-Shirt Size 估算

- 为该 Epic 提供高层级的 T-Shirt Size 估算（例如 S、M、L、XL）。
