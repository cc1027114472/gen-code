# 站点模式

这个文件夹用于存放经过验证、可复用的特定域名浏览备注。

只有当某个模式已经在真实使用中被确认后，才在这里新增文件。目的是保留那些能帮助后续会话避免重复试错的事实，而不是收集猜测。

## 工作流

1. 如果目标域名已经有备注，在开始浏览前先读它。
2. 把备注当作强提示，而不是绝对真相。
3. 如果你确认了新的可复用模式，就在任务结束后更新该备注。
4. 只在事实、已验证模式和已确认陷阱被验证后再写入。

## 这里应该放什么

- 与访问相关、经过验证的 URL 或导航模式
- 已知的登录或会话要求
- 针对该站点已经证明有效的提取策略
- 已知陷阱、假错误状态或脆弱的交互模式

## 这里不应该放什么

- 未确认的猜测
- 只在某一次成立、无法重复验证的站点备注
- 已经属于主 skill 的通用浏览建议

## 文件模板

每个域名创建一个文件：

```markdown
---
domain: example.com
aliases:
  - Example
updated: 2026-03-28
---

## Platform Traits

关于渲染、认证、导航或反自动化行为的事实。

## Proven Patterns

已经验证有效的具体路径、选择器、工作流或提取技巧。

## Known Traps

那些看起来像产品行为、但实际上是访问路径问题、过期状态或误导性错误的情况。
```

## 编写规则

- 保持条目简短且基于事实。
- 优先写可复现的事实，而不是建议。
- 当你对备注做出实质性修订时，更新日期。
