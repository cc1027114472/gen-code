# QA 问题分类

## 严重级别

| 严重级别 | 定义 | 示例 |
|----------|------|------|
| **critical** | 阻断核心流程、造成数据丢失或直接让应用崩溃 | 表单提交后报错页、结账链路断掉、删除操作没有确认且造成数据消失 |
| **high** | 主要功能不可用或几乎不可用，且没有可接受的绕路方案 | 搜索结果错误、文件上传静默失败、登录后死循环跳转 |
| **medium** | 功能能用，但存在明显问题，通常还有绕路方案 | 页面加载过慢、缺少校验但仍能提交、只在移动端破版 |
| **low** | 细节和观感问题，不影响主流程 | 文案错字、1px 对齐问题、hover 状态不一致 |

## 分类

### 1. Visual / UI

- 布局破裂、元素重叠、文字被裁切、出现横向滚动条
- 图片缺失或损坏
- z-index 错乱
- 字体、颜色、间距风格不一致
- 动画卡顿或过渡中断

### 2. Functional

- 链接跳错页或 404
- 按钮点击没有反应
- 表单校验缺失、错误或可绕过
- 状态不持久、刷新后数据丢失
- 搜索结果错误或为空

### 3. UX

- 导航令人困惑，或者进入死胡同
- 缺少加载反馈
- 错误消息过于含糊
- 危险操作没有确认
- 交互模式前后不一致

### 4. Content

- 错别字、语法问题
- 文案过期或与当前功能不符
- 残留占位文案
- 文本截断没有合理处理
- 空状态提示无意义

### 5. Performance

- 页面加载超过 3 秒
- 滚动卡顿
- 明显布局抖动
- 单页网络请求过多
- 图片体积过大或未优化

### 6. Console / Errors

- JavaScript 异常
- 4xx / 5xx 请求失败
- CORS 或 CSP 问题
- mixed content 警告
- 其他可能预示未来问题的 console 警告

### 7. Accessibility

- 图片没有 alt 文本
- 表单控件没有标签
- 键盘无法导航
- 焦点被困在局部区域
- ARIA 属性缺失或错误
- 对比度过低

## 每页检查表

1. 视觉扫描，确认布局、文字、图片和对齐。
2. 点击所有核心交互元素，确认它们真的做了该做的事。
3. 测表单，包含空提交、非法输入和边界输入。
4. 测导航，确认有进有出，没有走进死胡同。
5. 测空态、加载态、错误态和溢出态。
6. 每次交互后重新检查 console。
7. 必要时检查移动端和平板 viewport。
8. 如果有登录态或角色差异，确认边界是否正确。
