# FORMS.md

本文档说明如何在 skill 中组织“表单填写”类场景的附加参考资料，目标是让主 `SKILL.md` 保持精简，同时在需要时为 Codex 提供稳定、可复用的操作细节。

## 适用场景

- 技能需要指导代理填写网页表单、桌面表单或结构化录入界面。
- 主文档只想保留入口规则，而把字段级说明拆到独立文件。
- 不同表单存在不同校验、默认值、错误提示或提交流程。

## 推荐内容结构

建议在 `FORMS.md` 中按表单类型分段记录以下信息：

1. 表单用途与触发条件。
2. 关键字段、字段含义与必填规则。
3. 默认值、约束条件、联动关系与异常处理。
4. 提交前检查项与提交后验证信号。

## 编写原则

- 保持字段名称、按钮名称、错误提示与真实界面一致。
- 只记录代理完成表单任务真正需要的细节，不要把主文档里的通用规则重复一遍。
- 若不同产品或模块的表单差异明显，按模块分节，而不是混在一张总表里。
- 如果表单规则会频繁变化，主 `SKILL.md` 只保留“需要时读取 `FORMS.md`”这一导航提示。

## 示例组织方式

```text
payments-skill/
├── SKILL.md
├── FORMS.md
└── references/
    └── field-dictionary.md
```

这种拆分方式可以让 Codex 先读主文档，再在真正遇到表单填写任务时按需加载字段细节。
