# DOCX-JS.md

本文档示例化说明：当某个 skill 需要围绕 `docx.js`、浏览器文档生成或前端导出链路工作时，如何把专项知识拆到独立参考文件。

## 建议记录的信息

- 所用库或运行环境的边界。
- 常见输入模板与输出产物。
- 影响兼容性的 API、版本或浏览器限制。
- 调试导出失败时要先检查的信号。

## 为什么单独拆出

这类内容通常技术细节较多，如果直接塞进主 `SKILL.md`，会让不相关任务也背上额外上下文负担。拆成单独参考文件后，只有在真正处理该主题时才需要读取。

## 编排建议

- 把稳定规则写在前面。
- 把环境差异和问题排查写在后面。
- 对命令、库名和 API 名保持原文，说明文字用中文。
