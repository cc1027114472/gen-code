# EXAMPLES.md

本文档用于存放可复用的任务示例，帮助 Codex 迅速理解某类 skill 应如何组织输入、输出和步骤。

## 何时适合拆出示例文件

- 主文档已经足够长，不适合继续堆放大量示例。
- 同一 skill 需要覆盖多个子场景。
- 示例本身是高价值模板，后续会被频繁复用。

## 推荐示例类型

1. 最小可用示例：展示最短路径。
2. 常见生产示例：覆盖真实约束、校验和失败处理。
3. 反例：说明不要怎么写或为什么会失败。

## 示例写法原则

- 保持与主工作流一致，不要引入另一套规则。
- 对命令、路径、字段名和输出格式尽量给出稳定示例。
- 若示例依赖额外上下文，先在开头写明前提。

## 一个简单模板

```markdown
### 示例：创建新 skill

场景：
- 用户要创建一个新 skill，并要求带 `agents/openai.yaml`

做法：
1. 先创建基础目录和 `SKILL.md`
2. 再生成或补齐 `agents/openai.yaml`
3. 最后运行校验脚本确认结构有效
```
