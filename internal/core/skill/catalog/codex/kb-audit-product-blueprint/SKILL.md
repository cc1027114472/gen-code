# KB Audit 产品蓝图

用于这套 `kb-audit -> 产品蓝图` 项目的总控型 skill。
当任务是继续推进、审查、收口、同步整套产品蓝图文档体系时，优先使用本 skill；如果只是做通用编码或通用页面设计，不使用本 skill。
先读 [references/source-of-truth.md](references/source-of-truth.md)。

## 本 Skill 覆盖什么
- `F:\codex\artifacts\kb-audit` 真源材料
- `F:\codex\docs\产品蓝图` 主产出目录
- 四系统边界与模块归属控制
- `01~10` 主模块文档体系
- 逐页面文档六件套与对应 `xlsx`
- 治理总表 `11~17`
- 完成定义、路线图、汇报文档 `18~19`

## 固定系统边界

不要偏离以下边界：
- `公司官网`：只做品牌表达、价值说明、转化入口、登录入口
- `租户后台`：只做内部业务主链、审批结果、经营汇总、报表、经营分析
- `劳务系统`：只做工人、班组、高频作业与外部协同
- `平台后台`：只做平台治理、租户治理支撑、集成、监控、补偿、审计

硬规则：

- `06-经营分析` 只做分析、下钻、复盘、回溯，不做业务录单、审批办理、付款登记
- `07-公司官网` 不承接业务表单、审批办理、经营看板
- `08-平台后台` 不默认下钻租户业务正文，不替代租户审批工作台
- `03-劳务高频作业` 只承接高频作业；审批结果、经营汇总、报表仍回租户后台查看

## 输出物分层

默认输出物分三层：

1. 结构与总览文档：`00~10`
2. 治理总表文档：`11~17`
3. 完成定义、路线图、汇报文档：`18~19`

其中模块级逐页产物默认包括：
1. `逐页面原型稿`
2. `逐页面功能清单`
3. `xlsx`
4. `逐页流程图`
5. `逐页字段说明`
6. `逐页字段联动说明`

## 标准工作顺序

1. 先从真源文档恢复上下文，不要只靠历史记忆
2. 先判断用户要改的是哪一层：
   - 系统/模块规划
   - 逐页面文档
   - 治理总表
   - 总览与汇报
3. 先更新最具体的文档，再同步聚合文档
4. 如边界、页面范围、规则口径发生变化，再检查 `11~19` 是否需要联动

## 固定写法规则

- 一律使用中文命名
- 一律从产品经理视角写，不写技术实现说明
- 页面名、模块名在所有逐页文档族中保持一致
- 尽量沿用项目既有口径，不另起一套命名体系

逐页面功能清单固定 10 列：

`一级模块 / 二级模块 / 功能 ID / 功能名称 / 功能描述 / 优先级 / 依赖功能 ID / 验收标准 / 需求来源 / 备注`

其中：
- `功能描述` 必须写清“做什么、谁做、为什么做、步骤怎么走”
- `验收标准` 必须写成 `1. 2. 3.` 这种可验收句式

## 涉及 XLSX 时

如果任务包含逐页面功能清单 `xlsx`，默认保持以下格式：

- 工作表：`PageFeatureList`
- 第 1 行：合并标题
- 第 2 行：副标题
- 第 3 行：表头
- 冻结：`A4`
- 第 3 行起开启筛选

除非用户明确要求修改，否则沿用现有项目样式。

## 每次完工都检查

1. 原型稿、功能清单、流程图、字段说明、字段联动说明中的页面命名是否一致
2. 页面归属是否串系统
3. 功能清单写法是否仍满足固定描述/验收要求
4. 聚合文档里是否还残留“待补”“建议层”“只覆盖部分模块”等过期说法
5. 汇报文档中的“已完成 / 下一步”是否与真实文件状态一致

## 何时继续下钻读取

遇到以下场景再下钻更多文档：

- 做逐页续写：读取对应模块文档
- 做权限、敏感字段、默认值、异常、下钻、回写：读取 `11~17`
- 做完成情况、路线图、汇报口径：读取 `18~19` 与总览文档

## 默认目标

这个 skill 的目标不是无限补散页，而是在保持边界稳定的前提下，让整套产品蓝图持续演进且保持一致。
