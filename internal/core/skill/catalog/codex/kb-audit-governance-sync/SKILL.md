# KB Audit 治理同步

用于同步这套蓝图中的治理总表，重点范围是 `11~17`：
- `11-页面动作级权限与敏感字段治理说明`
- `12-逐页权限补充规范`
- `13-角色包与标准授权矩阵总表`
- `14-模块级页面动作权限矩阵总表`
- `15-敏感字段分级与脱敏规则总表`
- `16-字段默认值与异常规则总表`
- `17-跨页下钻与回写一致性总表`

## 先看什么
先读：
- `F:\codex\docs\产品蓝图\11~17`
- 当前要同步的模块逐页文档
- `F:\codex\artifacts\kb-audit\角色模型与权限边界说明.md`
- `F:\codex\artifacts\kb-audit\审批中心产品设计方案.md`

不要脱离逐页文档凭空写总表。

## 核心目标

治理总表的作用不是新增业务页面，而是把已经存在的页面和规则统一起来。
默认要解决五类问题：

1. 谁能进？
2. 谁能看？
3. 谁能做？
4. 哪些字段能看全量？
5. 跨页之后是否还能做？

## 固定边界

- `06-经营分析`：只做分析、下钻、复盘、回溯
- `07-公司官网`：只做品牌、转化、登录分流
- `08-平台后台`：只做治理、连接、监控、补偿、审计
- `03-劳务高频作业`：审批结果、经营汇总、报表仍回租户后台

## 同步顺序

建议顺序：
1. 先找逐页文档里的真实规则
2. 再改模块级或全局总表
3. 最后检查总表是否还残留旧阶段表述

## 常见同步点

- 模块已经补齐，但 `14/16/17` 还写成“建议层”
- 模块已经全量覆盖，但总表里还写“只覆盖部分模块”
- 页面规则改了，但敏感字段、默认值、异常、回写规则没同步
- 汇总结论和真实文件状态不一致

## 每次都检查

1. 总表里不能保留“待补”“建议层”“只覆盖部分模块”之类旧口径
2. 总表结论必须和当前 `01~10` 的真实完成状态一致
3. 总表不能反向新增业务动作
4. 总表不能把分析页写成执行页
5. 总表不能把平台页写成租户正文页
