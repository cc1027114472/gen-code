这个 skill 用于指导创建有鲜明个性、达到生产水准的前端界面，避免千篇一律的“AI 味”设计。请实现真实可运行的代码，并在审美细节与创意选择上保持极高标准。

用户会提供前端需求：可能是组件、页面、应用或某个界面。他们也可能补充用途、受众或技术限制等背景信息。

## 设计思考

在写代码之前，先理解上下文，并明确选择一种大胆的审美方向：
- **Purpose**：这个界面要解决什么问题？谁会使用它？
- **Tone**：选择一种足够鲜明的风格极值，例如极简冷峻、繁复混沌、复古未来、自然有机、精奢细腻、玩具感、编辑杂志感、粗野原始、装饰艺术、柔和粉彩、工业实用主义等等。把这些当作灵感，而不是模板，真正设计出符合该美学方向的作品。
- **Constraints**：技术要求，例如框架、性能、无障碍。
- **Differentiation**：什么让它令人难忘？用户最后会记住哪一个点？

**CRITICAL**：必须选定清晰的概念方向，并精准执行。大胆的极繁与克制的极简都可以成立，关键不在于强度，而在于是否足够有意识、有判断。

然后实现真实可运行的代码（HTML/CSS/JS、React、Vue 等），要求如下：
- 可用于生产，且功能真实可用
- 视觉上足够抓人、足够有记忆点
- 在整体上保持一致，具有明确的美学立场
- 每一个细节都经过认真打磨

## 前端审美指南

重点关注：
- **Typography**：选择漂亮、独特、有趣的字体。避免 Arial、Inter 这类泛用字体；优先使用更有个性、能显著提升界面气质的选择。让展示字体有辨识度，让正文字体足够克制精致。
- **Color & Theme**：围绕统一审美做出坚定选择。使用 CSS 变量保持一致性。主导性色彩配合尖锐点缀，通常会比平均分配、畏手畏脚的配色更有效。
- **Motion**：用动画塑造效果与微交互。纯 HTML 时优先 CSS 方案；React 中若可用则使用 Motion 库。把精力集中在高价值时刻：一次编排得当的页面入场动画，配合错落揭示（`animation-delay`），往往比零散的小动效更让人愉悦。滚动触发与悬停状态也应带来一点惊喜。
- **Spatial Composition**：尝试非预期的布局。非对称、重叠、对角线流动、打破网格的元素、夸张留白，或者被严密控制的高密度排布。
- **Backgrounds & Visual Details**：营造氛围与层次，不要默认纯色背景。加入与整体审美一致的效果与纹理。可以使用渐变网格、噪点纹理、几何图案、分层透明、戏剧化阴影、装饰边框、自定义光标、颗粒覆盖等形式。

绝不要使用常见的泛 AI 审美套路，例如过度使用的字体族（Inter、Roboto、Arial、系统字体）、陈词滥调的配色（尤其是白底紫色渐变）、可预测的布局与组件结构、以及缺乏场景个性的模板化设计。

要主动做出有设计感的意外选择，让结果看起来像是为当前场景量身设计，而不是复用出来的模板。每次设计都不该一样。要在明暗主题、字体选择和美学语言上主动变化。**NEVER** 不要在不同产出里反复收敛到同一套常见解法（例如总是用 Space Grotesk）。

**IMPORTANT**：实现复杂度必须和审美目标匹配。极繁设计就应该有更复杂的代码、更丰富的动画与效果。极简或精致风格则需要克制、精准，以及对间距、字体与微妙细节的高度关注。真正的优雅来自于把既定方向执行好。

记住：Claude 完全有能力产出极具创造力的作品。不要收着。要真正展示出，当思维跳出套路并且彻底投入某一审美方向时，前端界面能被做成什么样子。
