# KB Audit Page Docs

用于这套项目的逐页文档产出与续写。
适用范围：
- `01~10` 模块包续写
- `逐页面原型稿`
- `逐页面功能清单`
- `逐页流程图`
- `逐页字段说明`
- `逐页字段联动说明`
- 对应 `xlsx`

## 先读哪些真源

先看这些，再写文档：

- `F:\codex\artifacts\kb-audit\业务流程图大白话总说明.md`
- `F:\codex\artifacts\kb-audit\产品模块理解与使用流程.md`
- `F:\codex\artifacts\kb-audit\审批中心产品设计方案.md`
- `F:\codex\artifacts\kb-audit\角色模型与权限边界说明.md`
- `F:\codex\artifacts\kb-audit\本项目功能清单_原型设计视图.xlsx`
- `F:\codex\artifacts\kb-audit\本项目功能清单主表.xlsx`

## 固定边界

- `公司官网`：只做品牌、价值说明、转化入口、登录入口
- `租户后台`：只做内部业务主链、审批结果、经营分析、报表查看
- `劳务系统`：只做工人、班组、高频作业与外部协同
- `平台后台`：只做平台治理、集成、监控、补偿、审计

硬规则：

- `06-经营分析` 不做录单、审批办理、付款登记
- `07-公司官网` 不写后台业务动作
- `08-平台后台` 不默认下钻租户业务正文
- `03-劳务高频作业` 只承接高频作业；审批结果、经营汇总、报表回租户后台看

## 逐页文档顺序

默认顺序：
1. 先写逐页面原型稿
2. 再写逐页面功能清单
3. 再补 `xlsx`
4. 再写逐页流程图
5. 再写逐页字段说明
6. 再写逐页字段联动说明

## 功能清单固定规则

固定 10 列：

`一级模块 / 二级模块 / 功能 ID / 功能名称 / 功能描述 / 优先级 / 依赖功能 ID / 验收标准 / 需求来源 / 备注`

功能描述必须写清：
- 功能做什么
- 谁来做
- 为什么做
- 操作步骤怎么走

验收标准必须写成：
- `1.`
- `2.`
- `3.`

这种可验收句式。

## xlsx 固定规则

- sheet：`PageFeatureList`
- 第 1 行：标题
- 第 2 行：副标题
- 第 3 行：表头
- 冻结：`A4`
- 筛选从第 3 行开始

除非用户明确要求改，否则沿用现有项目格式。

## 每次完工都检查

1. 页面命名在六类逐页文档里必须一致
2. 页面范围不能串系统
3. 功能清单写法必须保持现有项目口径
4. `md` 和 `xlsx` 必须保持一致
5. 不为了补页面破坏已经稳定的主结构
