# REFERENCE.md

本文档用于放置 skill 的方法清单、命令参考、参数说明或接口约束，供 Codex 在执行特定任务时按需读取，而不是把所有细节都堆进 `SKILL.md`。

## 适合放在这里的内容

- 命令或脚本的参数表。
- 输入输出格式说明。
- 常见错误与诊断提示。
- 需要精确复述的接口、schema 或配置字段。

## 编写建议

- 按主题或命令分节，保证检索路径清晰。
- 对需要逐字准确的字段名、参数名与文件路径保持原样。
- 若某段内容很长，可在小节开头补一行“何时需要读这一节”。

## 与主文档的关系

- `SKILL.md` 负责触发条件、工作流和总导航。
- `REFERENCE.md` 负责细节真值。
- 当任务只需要总体流程时，不必自动加载本文件。

## 最佳实践

在主文档中用类似下面的提示引导按需读取：

```markdown
- 需要参数细节时，读取 [REFERENCE.md](REFERENCE.md)
```
